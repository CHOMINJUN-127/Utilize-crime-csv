# 범죄 데이터 기반 딥러닝 분석 프로젝트
공공 범죄 CSV 데이터를 전처리하고 딥러닝 모델을 통해 범죄 발생 패턴을 분석하는 프로젝트


## 📌 프로젝트 개요
AI 기술의 발전과 함께 범죄의 형태 또한 복잡해지고 있어, 경험이나 직관에 의존한 분석의 한계를 보완할 필요가 있다고 판단하였다. 이에 따라 경찰청에서 제공하는 범죄 발생 장소별 통계 데이터를 활용하여, 장소 특성과 범죄 유형 간의 관계를 분석하고자 하였다. 본 프로젝트에서는 다수의 변수를 동시에 학습할 수 있는 딥러닝 모델을 적용하여 특정 환경에서 발생 가능성이 높은 범죄 유형의 패턴을 분석하였다. 이를 통해 AI가 범죄 데이터를 학습하고 분류하는 과정을 직접 확인하고, 범죄 예방 및 치안 분야에서의 활용 가능성을 탐구하고자 하였다.


## ✨ 주요 기능

- 범죄 발생 장소별 통계 CSV 데이터를 기반으로 고차원 범죄 패턴을 학습하는 다중 분류 딥러닝 모델 구현
- 장소별 범죄 발생 건수(약 68개 Feature)의 조합 패턴을 입력으로 사용하여 범죄 대분류를 예측
- 결측치, 레이블 불균형, 스케일 차이 등 범죄 통계 데이터의 구조적 문제를 고려한 전처리 파이프라인 구성
- 학습 과정에서 정확도(Accuracy)와 손실(Loss)의 변화를 시각화하여 모델의 학습 안정성과 과적합 여부를 확인
- 테스트 데이터에 대한 예측 결과를 혼동 행렬과 분류 리포트로 분석하여 범죄 유형별 분류 성능을 비교


## 🧠 시스템 구조

본 프로젝트는 범죄 발생 장소별 통계 데이터를 입력으로 받아  
전처리 → 딥러닝 학습 → 성능 평가 및 시각화의 단계로 구성된 분석 파이프라인을 따른다.

1. **데이터 입력 단계**
   - 경찰청에서 제공한 범죄 발생 장소별 통계 CSV 파일을 사용
   - 한글 인코딩(cp949, euc-kr)을 고려하여 데이터 로드
   - 장소별 범죄 발생 건수를 수치형 Feature로 구성

2. **데이터 전처리 단계**
   - 결측치는 범죄 미발생(0건)으로 간주하여 0으로 처리
   - 단일 샘플만 존재하는 범죄 대분류 클래스 제거하여 학습 오류 방지
   - 범죄 대분류를 Label Encoding을 통해 정수 레이블로 변환
   - 장소별 발생 건수 간 스케일 차이를 줄이기 위해 StandardScaler로 정규화 수행

3. **모델 학습 단계**
   - 입력층–은닉층–출력층 구조의 다층 퍼셉트론(MLP) 기반 딥러닝 모델 설계
   - ReLU 활성화 함수와 Dropout을 적용하여 비선형 패턴 학습 및 과적합 방지
   - Softmax 출력층을 통해 범죄 대분류 다중 분류 수행
   - 학습 데이터와 검증 데이터를 분리하여 모델 성능을 반복적으로 검증

4. **성능 평가 및 시각화 단계**
   - 학습 과정에서 정확도(Accuracy)와 손실(Loss) 변화 시각화
   - 테스트 데이터에 대한 예측 결과를 기반으로 분류 리포트 생성
   - 혼동 행렬(Confusion Matrix)을 시각화하여 범죄 유형별 분류 성능 분석
   - 히트맵을 통해 범죄 대분류와 발생 장소 간의 상관관계 탐색

이러한 구조를 통해 범죄 데이터를 단순 통계가 아닌  
딥러닝 기반 패턴 분석 대상으로 해석하고,  
AI가 장소별 범죄 특성을 어떻게 학습하는지를 단계적으로 확인할 수 있도록 설계하였다.

### 시각화

- **Seaborn**  
  범죄 유형별 발생 건수 분포, 혼동 행렬, 히트맵 시각화

### 모델 평가
- **scikit-learn**
  - `classification_report` : 정확도, 정밀도, 재현율 등 분류 성능 평가  
  - `confusion_matrix` : 범죄 대분류별 예측 결과 비교 분석

## 🔍 문제 해결 과정

### 1. 데이터 구조에 따른 분석 한계 인식
경찰청 범죄 발생 장소별 통계 데이터는 범죄 대분류별로  
다수의 장소 변수를 포함하는 고차원 수치 데이터로 구성되어 있다.  
이러한 데이터는 단순 평균 비교나 규칙 기반 분석만으로는  
장소 간 복합적인 패턴을 파악하는 데 한계가 있다고 판단하였다.

→ **해결 방향**  
여러 장소 변수 간의 비선형적 관계를 동시에 학습할 수 있는  
딥러닝 기반 분류 모델을 적용하기로 결정하였다.

---

### 2. 결측치 및 통계 데이터 특성 문제 해결
범죄 통계 데이터에는 일부 장소에서 발생 건수가 기록되지 않은 항목이 존재하였다.  
해당 결측치는 데이터 누락이라기보다 특정 장소에서 범죄가 발생하지 않은 경우로  
해석하는 것이 타당하다고 판단하였다.

→ **해결 방법**  
결측치를 제거하지 않고 0으로 처리하여  
장소별 범죄 발생 여부를 그대로 학습 데이터에 반영하였다.

---

### 3. 클래스 불균형 및 학습 오류 문제 해결
범죄 대분류 중 일부 항목은 데이터 개수가 1건뿐이어서  
학습·검증 데이터 분리 과정에서 오류를 발생시켰다.  
이는 모델 성능 문제가 아닌 데이터 구조로 인한 학습 불가능 상태였다.

→ **해결 방법**  
단일 샘플만 존재하는 범죄 대분류를 사전에 제거하여  
모든 클래스가 최소 2개 이상의 샘플을 갖도록 데이터 구조를 재구성하였다.

---

### 4. 장소별 수치 스케일 차이 문제 해결
장소별 범죄 발생 건수는 변수마다 값의 범위 차이가 매우 컸으며,  
이 상태로 학습할 경우 특정 장소 변수에 모델이 과도하게 영향을 받을 가능성이 있었다.

→ **해결 방법**  
StandardScaler를 적용하여 모든 입력 변수를 정규화함으로써  
모델이 장소별 상대적 패턴에 집중하도록 유도하였다.

---

### 5. 모델 과적합 방지 및 성능 검증
딥러닝 모델은 학습 데이터에 과도하게 적합될 경우  
새로운 데이터에 대한 예측력이 저하될 위험이 있다.

→ **해결 방법**  
Dropout을 적용한 다층 퍼셉트론(MLP) 구조를 설계하고,  
검증 데이터 정확도 및 손실 변화를 함께 확인하여  
과적합 여부를 지속적으로 점검하였다.

---

### 6. 결과 해석 및 한계 인식
혼동 행렬과 분류 리포트를 통해  
범죄 유형별 예측 성능의 차이를 확인할 수 있었으며,  
일부 범죄 유형에서는 데이터 분포 불균형으로 인한 오분류가 발생하였다.

이를 통해 딥러닝 모델의 성능은  
데이터 양과 분포에 크게 의존한다는 점을 확인하였고,  
향후 데이터 확장 및 변수 다양화의 필요성을 인식하게 되었다.


## 📊 모델 성능 평가

딥러닝 모델의 성능 평가는 학습 과정 시각화와 테스트 데이터 기반 평가를 통해 수행하였다.

### 1. 학습 과정 평가
모델 학습 과정에서 학습 데이터와 검증 데이터의 정확도(Accuracy)와 손실(Loss)을 함께 관찰하였다.  
그 결과, 에포크가 증가함에 따라 학습 정확도와 검증 정확도가 함께 상승하였으며,  
손실 값 또한 안정적으로 감소하는 경향을 보였다.

이를 통해 모델이 학습 데이터에만 과도하게 적합되지 않고  
일정 수준의 일반화 성능을 유지하며 학습되었음을 확인하였다.

---

### 2. 테스트 데이터 성능 평가
학습이 완료된 모델을 테스트 데이터에 적용하여 최종 성능을 평가하였다.

- 정확도(Accuracy): 테스트 데이터 기준으로 범죄 대분류를 일정 수준 이상 정확하게 분류함
- 손실(Loss): 학습 과정에서 과도한 진동 없이 안정적인 값을 유지함

해당 결과는 장소별 범죄 발생 패턴이  
딥러닝 모델을 통해 의미 있는 분류 정보로 학습될 수 있음을 보여준다.

---

### 3. 혼동 행렬(Confusion Matrix) 분석
혼동 행렬을 통해 범죄 대분류별 예측 결과를 시각적으로 분석하였다.

- 일부 범죄 유형은 비교적 높은 정확도로 분류되었으며,
- 발생 건수가 적거나 장소 특성이 유사한 범죄 유형 간에는 오분류가 발생하였다.

이는 범죄 데이터의 클래스 불균형과  
장소 변수 간 중복된 특성이 모델 예측에 영향을 미친 결과로 해석할 수 있다.

---

### 4. 분류 리포트 기반 해석
분류 리포트를 통해 각 범죄 유형별 정밀도(Precision), 재현율(Recall), F1-score를 확인하였다.  
이를 통해 단순 정확도 수치뿐만 아니라  
범죄 유형별 예측 성능의 차이를 정량적으로 비교할 수 있었다.

특정 범죄 유형에서 재현율이 낮게 나타난 것은  
해당 범죄의 데이터 수가 상대적으로 적거나  
다른 범죄 유형과 장소 패턴이 유사하기 때문으로 판단된다.

---

### 5. 성능 평가를 통해 얻은 시사점
본 모델은 범죄 발생 장소별 통계 데이터만을 활용했음에도  
범죄 유형 간의 일정한 분류 성능을 보였다는 점에서 의미가 있다.  
동시에 데이터 분포와 변수 구성에 따라  
딥러닝 모델 성능이 크게 달라질 수 있다는 한계 또한 확인하였다.


### 🚀 향후 개선 사항

1. **데이터 범위 확장**
   현재 분석은 범죄 발생 장소별 통계 데이터에 한정되어 있다.  
   향후 범죄 발생 시간대, 요일, 지역 정보 등의 추가 변수를 결합한다면  
   범죄 유형 예측의 정밀도를 더욱 향상시킬 수 있을 것으로 기대된다.

2. **클래스 불균형 문제 개선**
   일부 범죄 대분류는 데이터 수가 적어 모델 예측 성능에 한계가 있었다.  
   향후 데이터 증강 기법 또는 클래스 가중치 조정을 적용하여  
   불균형으로 인한 오분류 문제를 완화할 수 있을 것이다.

3. **모델 구조 고도화**
   현재는 다층 퍼셉트론(MLP) 기반 모델을 사용하였으나,  
   향후에는 Attention 메커니즘이나 앙상블 모델을 적용하여  
   장소 변수 간 중요도를 보다 명확하게 반영하는 모델을 설계할 수 있다.

4. **설명 가능한 AI(XAI) 적용**
   딥러닝 모델의 예측 결과를 해석하기 위해  
   SHAP, LIME과 같은 설명 가능한 AI 기법을 도입한다면  
   특정 범죄 유형 예측에 영향을 준 주요 장소 요인을 분석할 수 있을 것이다.

5. **범죄 예방 관점의 활용 가능성 탐구**
   본 분석 결과를 범죄 예방 및 치안 정책 수립 관점에서 활용하기 위해  
   고위험 장소 유형을 사전에 식별하는 방식으로 확장할 수 있다.  
   이를 통해 AI 기반 범죄 예측 모델의 사회적 활용 가능성을 탐구할 수 있을 것이다.
